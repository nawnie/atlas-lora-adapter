## Canonical distinction

- **CLI** is for direct operation, smoke tests, server startup, model pulls, and local debugging.
- **API** is for apps, automation, agents, Gradio wrappers, and repeatable workflows.
- **Guide inserts** are human teaching material.
- **Atlas cards** route, evaluate, and source that material.

## Source policy

For API and CLI instructions, prefer official documentation over blog/tutorial material. Commands change, flags drift, and the internet occasionally hands you a cursed shell incantation wearing a helpful hat.
