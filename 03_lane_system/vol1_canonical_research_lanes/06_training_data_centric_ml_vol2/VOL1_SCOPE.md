# Vol. 1 Scope — Training, Data-Centric ML, and Volume II Research Spine

## Lane thesis

    Training knowledge should start with data, objective, evaluation, and failure evidence before discussing trainer buttons or model hype.

## In scope

- dataset engineering
- LoRA/fine-tuning ladders
- trainer landscape
- evaluation methods
- leakage and split discipline
- label quality
- consumer hardware limits
- training ethics/law

## Out of scope

- pretending every user needs full fine-tuning
- publishing dataset advice without legal/consent boundaries
- confusing prompting, RAG, LoRA, and full training

## Vol. 1 success criteria

This lane is Vol. 1 balanced when a local assistant can:

- answer beginner-to-intermediate questions from this lane without confusing it with adjacent lanes;
- cite or point to the original source family/provenance path;
- retrieve exact technical names when exact names matter;
- distinguish stable principles from fast-moving implementation details;
- identify when a user needs a local schema check, install check, or source refresh before acting.
