# Report Template Policy

Status: active policy  
Version added: v0.2.9-dev17  
Date: 2026-05-22

## Purpose

Report templates are allowed and encouraged.

They help make future results easier to record.

But templates must not read like completed results.

## Required template language

Templates should use:

```text
TBD
not run yet
pending local install
pending training run
pending evaluation
```

Templates should avoid:

```text
passed
improved
validated
successful
works
```

unless the field is clearly an example or placeholder.

## Template rule

A template is a container.

A run log is evidence.

Do not confuse the two.

## Current template files

Known useful templates include:

```text
05_evaluation/SMOKE_TRAIN_RESULTS_REPORT_TEMPLATE.md
04_training_data/RUNNABLE_TRAINING_LAYER.md
04_training_data/LAB_INSTALL_GUIDE.md
```

These should remain in the repo because they define how future results will be captured.
