# Canonical AIWF Terminology

Use these names consistently in active adapter-facing documents.

| Concept | Canonical wording | Notes |
|---|---|---|
| Parent project | AI Without Fear | Use full name on first mention. |
| Abbreviation | AIWF | Use after the full name is clear. |
| Human guide | AI Without Fear Field Guide | Human-readable teaching layer. |
| Machine archive | AIWF Atlas Vol. 1 | Machine-facing domain memory layer. |
| Short machine name | Atlas Layer | Acceptable after first use. |
| Retrieval system | Atlas system | Use when discussing retrieval-augmented systems generally. |
| Source material | source payload / provenance source | Raw source material is evidence, not final answer style. |
| Current lane | canonical research lane | Curated machine-facing summary layer. |
| Future work | roadmap item / expansion queue | Future-work markers belong only in `07_EXPANSION_QUEUE/`. |

## Avoid in active canonical language

- “unstructured knowledge pile”
- “unstructured archive”
- “final all-purpose reference”
- “permanently validated”
- “best model” without source and timeframe
- “industry standard” without source and timeframe
- AI-draft provenance scaffolding language
- unfinished serial-delivery language

## Allowed but context-limited

- “dataset” only when referring to actual data-science datasets, not the whole adapter.
- “training” only when referring to model/data training topics, not implying the adapter itself is a model training artifact.
- “Vol. 1” for display; `VOL1` may remain in file/folder/archive names for compatibility.


## Humor and mascot terminology

| Concept | Canonical wording | Notes |
|---|---|---|
| Humor voice | AIWF field-manual humor | Dry, practical, lightly absurd; never replaces accuracy. |
| Mascot/helper | Torchie | Optional helper/mascot for callouts and warnings. |
| Core positioning | force multiplier | AI as leverage, not replacement. |
| User journey | panic to participation | Use for teaching/brand framing. |
| Bad tooling chaos | dependency gremlin / workflow spaghetti | Allowed light humor for troubleshooting. |

## Avoid humor terms in serious contexts

Do not use mascot or joke language in license, consent, safety, security, or source-verification warnings unless the warning remains completely clear.


## v2.8 Brand Terms

Preferred terms:

- **Atlas Layer** — machine memory layer for AI systems.
- **Field Guide** — human teaching layer.
- **Force multiplier** — AI extends human reach; it does not replace human judgment.
- **Human control surface** — a UI layer, often Gradio, that lets people inspect and steer AI systems.
- **Source gate** — a rule that prevents unsupported claims from becoming canonical.
- **Failure signature** — a recognizable pattern that links symptoms to likely causes and fixes.
