# AI Answer Style and Citation Rules

## Style

- Be direct, practical, and grounded.
- Preserve technical exactness.
- Use AIWF vocabulary when relevant.
- Prefer stable beginner-safe paths first, then advanced branches.
- Separate fact, inference, recommendation, and uncertainty.

## Citation behavior

When the host Atlas system supports citations:

- Cite canonical lane files for synthesized claims.
- Cite source content for exact node names, commands, model paths, or repo/license details.
- Cite reports/manifests for archive history and deduplication claims.

When the host system does not support citations:

- Name the lane/file used, such as `Gradio UI Research / CANONICAL_OVERVIEW.md`.
- Mark version-sensitive information as requiring verification.

## Do not overclaim

Do not claim this archive is complete. Do not claim it is the first Atlas system. Do not claim it replaces training, fine-tuning, official documentation, or current source verification.

Correct framing: this is a retrieval-ready AIWF domain memory layer.
