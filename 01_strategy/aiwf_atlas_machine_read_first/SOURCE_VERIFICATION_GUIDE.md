# Source Verification Guide

This adapter is source-preserving by design. Research value comes from keeping the trail back to original papers, docs, repos, model cards, package pages, and user-supplied notes.

## Machine Use

1. Prefer canonical research lanes for synthesis.
2. Prefer primary/official sources for factual claims.
3. Treat model rankings, hardware claims, API behavior, and package instructions as fast-moving.
4. Use source verification manifests before answering high-confidence technical questions.
5. If a claim is in the volatile review queue, qualify the answer or verify externally before using it as current truth.

## Important Rule

Do not flatten source-backed content into unsourced summaries. Summaries are useful for retrieval; sources make the knowledge auditable.
