Short name:

```text
AIWF Atlas Reader LoRA
```

Repository/project family:

```text
AI Without Fear
```

## One-line description

AIWF Atlas Reader LoRA is a small-model research lab testing whether a LoRA can learn how to read a versioned structured-RAG Atlas without memorizing the Atlas itself.

## Plain-language description

AI Without Fear is about making AI systems understandable, local-first, practical, and less intimidating.

Atlas Reader LoRA fits that mission by turning a pile of AI troubleshooting knowledge into a map:

```text
lanes
cards
source rules
context packs
evaluation reports
```

The model is not asked to be a wizard.

It is asked to read the map without driving into a crater labeled “probably fine.”

## Brand principles

Use language that is:

- practical;
- source-aware;
- friendly;
- honest about limits;
- research-minded;
- useful to local builders;
- a little dryly funny when appropriate.

Do not use language that is:

- hype-driven;
- mystical;
- overconfident;
- vendor-worshipping;
- “trust me bro” disguised as a benchmark.

## AIWF tone

Good tone:

```text
Here is what this does.
Here is what it does not do.
Here is how we will test it.
Here is what failure would teach us.
```

Bad tone:

```text
This changes AI forever.
Tiny models are now solved.
The adapter is magic.
```

## Visual language

Preferred visual concepts:

- map;
- routing board;
- lanes;
- cards;
- dashboard;
- field manual;
- lab bench;
- test rig.

Avoid visual concepts that imply:

- superintelligence;
- magic;
- black-box authority;
- fake academic certainty.

## Naming conventions

Use:

```text
AIWF Atlas Reader LoRA
AIWF Atlas Contract
AIWF Atlas Lab
AIWF Training Lab
```

Do not rename the technical artifact away from:

```text
Atlas Reader LoRA
```

The AIWF brand wraps the project.

The technical name stays stable.

## Claim boundary

Safe brand claim:

```text
AIWF Atlas Reader LoRA is a research lab for testing structured-RAG reading behavior in small models.
```

Unsafe brand claim:

```text
AIWF Atlas Reader LoRA proves small models can replace larger models.
```

## Source policy

AIWF branding does not relax the source policy.

AI assistants remain drafting aids, not sources.

Performance claims require measured local outputs.

## Pre-install / no-results guard

This project has not yet been installed on the target PC.

No adapter has been trained.

No evaluation result exists yet.

Prepared scripts, templates, and diagrams are not wins; they are the test bench.


## Research identity

Use this as an AIWF research principle:

```text
Unknown does not mean impossible.
Unknown means instrument it.
```

This is not a claim that the idea works.

It is a commitment to run the test instead of pretending certainty.
